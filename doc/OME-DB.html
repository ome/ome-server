Table: public.analysis_path_map

This table links a path through a "tree" of analyses to specific analyses. Paths traverse the directed graph of analyses (or "tree" of analyses) from the root node to a specific leaf node. Due to branching and merging in this analysis "tree", analyses may belong to more than one path. A given path is composed of many analyses. This table specifies the many-to-many relationship between paths and analyses.

Table: public.features

The FEATURES table is used for ALL image features reguardless of algorithm. Image features are understood to be objects of interest within the image - blobs, cells, nuclei, etc. The properties of a feature are that it has a variable list of attributes. The kinds of attributes in this list and their values define everything known about the feature. Feature attribute tables contain a column named feature_id that refers to the attribute_id column of this table. The FEATURES table itself is technically an image attribute table because it contains an image_id. Algorithms that define features write rows into this table - one row per feature.

public.features Structure
F-Key Name Type Description
attribute_id serial oidNOT NULL PRIMARY KEY
public -> images image_id oid
name character varying(128)

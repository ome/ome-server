II-2. Downloading from CVS
 

The whole OME source code and other related artifacts can be freely downloaded (refer to Section III-3 for licensing issues) from the CVS repository located at SourceForge. To do so, you will need a CVS client. This is normally provided under Linux, however you can get one from http://www.cvshome.org/.
Before you start downloading from CVS, make a new directory and move there. Then, type the following at the command line prompt:

cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/ome login

When prompted for a password for anonymous, simply press the enter key. Then, type:

cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/ome co OME

The above will download the whole OME module in the current directory. What you get is the latest version of the files on CVS.

   Downloading the most recent files
 
 

However, if you're not interested in contributing to the OME development by writing code or testing the latest developed software, and you only want to build and deploy the build that this document refers to, you must type the following instead:

cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/ome co -r <build_id> OME

where <build_id> is the one reported on the cover page of this document. This will download the source code from CVS that was tagged as build_id.
For example, assuming the Document ID you find on the cover page is build-2003022819, then type:

cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/ome co -r build-2003022819 OME

Notice that a normal check out (that is, you don't specify -r and build_id) would download the latest version of the source code, which might not be the same as the tagged version which this document refers to.

   Retrieving a tagged build
 
 
If you retrieved this document by checking out the latest version on CVS (without specifying -r and build_id), then you got the SBDD that refers to the latest tagged build.   Latest build
 
 
If you want to build and deploy a previous build, then you should first take a look at the Tags History and see what the previous builds are. If you have already downloaded the latest OME root module from CVS, then go to the OME/doc/TagsHistory directory. Every entry is a text file that has the same name as the build it refers to and explains why that build was noteworthy to the developers.
If you don’t have the latest OME root module, you could retrieve it as explained above and then browse the TagsHistory directory. However, it may be more convenient to browse this directory directly on the CVS repository through its Web interface. To do so, just point your Web browser to:

http://cvs.sourceforge.net/cgi-bin/viewcvs.cgi/ome/OME/doc/TagsHistory/

When you have decided on which build, retrieve the relative OME root module as explained above. We recommend that you go through the relative SBDD, as dependencies and instructions may vary from build to build.

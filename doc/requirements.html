OS Requirements:

There is no requirement for any kind of UNIX per se - only certain open-source libraries, perl modules, packages, and a unix-like file-system.
To my knowledge, there is no single UNIX distribution that has all of these, but I may be wrong.
OME is known to run on many Linux distros, and Mac OS X (which is my development platform) - as long as the following dependencies are met.

Major package requirements:

  • Perl >= 5.6
  • Apache+mod_perl >= 1.3.12
  • PostgreSQL >= 7.1 with ODBC support (JDBC, Tcl/Tk optional but recommended)
  • ImageMagick+PerlMagick

Other Perl modules:

  • Storable
  • Apache::Session::File
  • Image::Magick (PerlMagick)
  • Frontier::XMLRPC (patched version here).
  • DBI (problems with 1.20. RH 7.1 ships with 1.14 which is too old - use the one here)
  • DBD::Pg (problems with many versions - use the one here)


All of the perl dependencies can be downloaded and installed with the following commands (as root): 

> cd OME/src/perl/
> ./perlCheck.pl
Note that you should execute this script from the directory into which you installed the OME perl scripts - the directory from which they will be served by Apache. See the install document. N.B.: This will not install postgres, Apache or mod_perl

Other Libraries:

Usually need to re-install libtiff if you want LZW support.  Get and install the following libraries:
tiff-v3.5.6-beta
libtiff-lzw-compression-kit

ImageMagick

To be useful, ImageMagick (and PerlMagick) need the following additional libraries, which are usually installed:
jpeg-6b
in the near future, also freetype-2.0.4 (this is latest version, can probably use an earlier one)
Installing required libraries on Mac OS X is greatly simplified with the use of Christoph Pfisterer's fink.
The perlCheck.pl script will install ImageMagick if it is missing, but it will not install these additional libraries.

PostgreSQL:

You can find out what version you have (if any) by typing:
> psql -V
It is best to compile PostgreSQL from its source - I have had stability problems with the binary RPMs. This may no longer be true with 7.1.3 - try binaries first if you want, then compile from source if they are unstable. PostgreSQL should be able to run for months with no problems. To compile PostgreSQL, you will need the following libraries (not strictly required, but highly recommended):
readline (from gnu.org - usually present on Linux distros, but not always)
ant (from the jakarta project - binary install or RPM is fine)

run the configure script with something like the following command: . 

./configure --with-perl --with-tcl --enable-odbc --with-java
You may also want to set the data directory using with the flag --datadir=DIRECTORY. Read the INSTALL document before doing anything. If you installed DBD::Pg before doing this, you must install it again (do a 'make clean' first!) so that it is linked against the proper Postgres libraries.
